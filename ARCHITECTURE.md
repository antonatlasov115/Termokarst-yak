# Архитектура проекта

## Обзор

Проект построен как Rust workspace с модульной архитектурой, где каждый крейт отвечает за свою область функциональности.

## Структура крейтов

```
thermokarst-yakutia/
├── Cargo.toml                 # Workspace конфигурация
├── README.md                  # Основная документация
├── ALGORITHM.md               # Описание алгоритма
├── EXAMPLES.md                # Примеры использования
├── .gitignore
│
└── crates/
    ├── core/                  # Базовые типы и трейты
    ├── physics/               # Физические расчеты
    ├── geology/               # Геологические процессы
    ├── simulation/            # Движок симуляции
    └── cli/                   # Интерфейс командной строки
```

## Крейт: thermokarst-core

**Назначение:** Базовые типы данных и интерфейсы

**Зависимости:**
- serde (сериализация)
- thiserror (обработка ошибок)

**Основные типы:**
- `SoilType` - типы грунтов (глина, песок, торф, суглинок, ил)
- `EnvironmentParams` - параметры окружающей среды
- `ThermokarstLens` - термокарстовая линза
- `SimulationResult` - результаты симуляции
- `ThermokarstStage` - стадии развития
- `ThermokarstError` - типы ошибок

**Файлы:**
```
core/
├── src/
│   ├── lib.rs          # Экспорт модулей
│   ├── types.rs        # Основные типы
│   └── error.rs        # Обработка ошибок
└── Cargo.toml
```

## Крейт: thermokarst-physics

**Назначение:** Физические расчеты термокарстовых процессов

**Зависимости:**
- thermokarst-core
- nalgebra (линейная алгебра)

**Модули:**
- `thaw` - расчет глубины протаивания (формула Стефана)
- `heat_transfer` - теплоперенос в грунте
- `subsidence` - просадка при оттаивании

**Основные компоненты:**
- `ThawDepthCalculator` - калькулятор протаивания
- `HeatTransferCalculator` - калькулятор теплопереноса
- `SubsidenceCalculator` - калькулятор просадки

**Файлы:**
```
physics/
├── src/
│   ├── lib.rs              # Экспорт модулей
│   ├── thaw.rs             # Протаивание
│   ├── heat_transfer.rs    # Теплоперенос
│   └── subsidence.rs       # Просадка
└── Cargo.toml
```

**Тесты:**
- Увеличение глубины протаивания со временем
- Влияние растительности на протаивание
- Зависимость просадки от льдистости
- Высокая просадочность торфа

## Крейт: thermokarst-geology

**Назначение:** Геологические процессы и анализ стабильности

**Зависимости:**
- thermokarst-core

**Модули:**
- `lateral_expansion` - боковое расширение
- `stability` - анализ стабильности

**Основные компоненты:**
- `LateralExpansionCalculator` - расчет диаметра и площади
- `StabilityAnalyzer` - оценка стабильности и рисков

**Файлы:**
```
geology/
├── src/
│   ├── lib.rs                  # Экспорт модулей
│   ├── lateral_expansion.rs    # Боковое расширение
│   └── stability.rs            # Анализ стабильности
└── Cargo.toml
```

**Тесты:**
- Рост диаметра со временем
- Влияние воды на расширение
- Определение стадий развития
- Оценка риска обрушения

## Крейт: thermokarst-simulation

**Назначение:** Движок симуляции и батч-обработка

**Зависимости:**
- thermokarst-core
- thermokarst-physics
- thermokarst-geology
- rayon (параллелизм)

**Модули:**
- `engine` - основной движок симуляции
- `batch` - параллельная обработка сценариев

**Основные компоненты:**
- `SimulationEngine` - движок симуляции
- `SimulationConfig` - конфигурация
- `BatchSimulator` - батч-обработка
- `DetailedSimulationResult` - детальные результаты

**Файлы:**
```
simulation/
├── src/
│   ├── lib.rs      # Экспорт модулей
│   ├── engine.rs   # Движок симуляции
│   └── batch.rs    # Батч-обработка
└── Cargo.toml
```

**Тесты:**
- Успешное выполнение симуляции
- Рост линзы со временем
- Батч-симуляция всех регионов

## Крейт: thermokarst-cli

**Назначение:** Интерфейс командной строки

**Зависимости:**
- thermokarst-core
- thermokarst-physics
- thermokarst-geology
- thermokarst-simulation
- clap (CLI парсинг)
- anyhow (обработка ошибок)

**Команды:**
- `simulate` - одиночная симуляция
- `batch` - батч-симуляция
- `analyze` - анализ результатов
- `config` - создание конфигурации

**Файлы:**
```
cli/
├── src/
│   ├── main.rs                 # Точка входа
│   └── commands/
│       ├── mod.rs              # Экспорт команд
│       ├── simulate.rs         # Команда simulate
│       ├── batch.rs            # Команда batch
│       ├── analyze.rs          # Команда analyze
│       └── config.rs           # Команда config
└── Cargo.toml
```

## Поток данных

```
1. Пользователь → CLI
   ↓
2. CLI → SimulationEngine
   ↓
3. SimulationEngine → Physics (ThawDepthCalculator, SubsidenceCalculator)
   ↓
4. SimulationEngine → Geology (LateralExpansionCalculator)
   ↓
5. SimulationEngine → Core (ThermokarstLens)
   ↓
6. SimulationEngine → Geology (StabilityAnalyzer)
   ↓
7. SimulationResult → CLI
   ↓
8. CLI → Пользователь (вывод/файл)
```

## Принципы дизайна

### 1. Модульность
Каждый крейт имеет четкую ответственность и может использоваться независимо.

### 2. Разделение ответственности
- **core** - данные
- **physics** - физика
- **geology** - геология
- **simulation** - оркестрация
- **cli** - интерфейс

### 3. Тестируемость
Каждый модуль содержит unit-тесты для проверки корректности расчетов.

### 4. Расширяемость
Легко добавить новые:
- Типы грунтов
- Региональные параметры
- Методы расчета
- Команды CLI

### 5. Производительность
- Параллельная обработка через rayon
- Эффективные структуры данных
- Release-оптимизация
