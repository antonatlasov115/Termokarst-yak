# Алгоритм расчета термокарста

## Формула Атласова для глубины протаивания

### Два режима расчета

Формула используется для двух разных физических процессов:

#### 1. Сезонная глубина протаивания (ALT - Active Layer Thickness)

Глубина сезонного оттаивания, которая каждый год оттаивает летом и замерзает зимой:

```
ξ_ALT = √(2λₜ·DDT_annual / (L·ρw·w^α)) · exp(β·(1-V)) · (1 + γ·ln(ΔT/ΔT₀)) · f_moisture
```

**Применение:**
- Расчет глубины активного слоя для одного сезона
- Валидация по полевым данным (Neleger, Churapcha)
- Время не участвует (t = 1 год)

**Типичные значения для Якутии:**
- Лес: 1.0-1.5 м
- Луг: 1.5-2.5 м
- Открытая тундра: 0.5-1.0 м

#### 2. Кумулятивная глубина термокарста (многолетний процесс)

Глубина термокарстового образования, которая накапливается за счет таяния подземных льдов:

```
ξ_thermo(t) = ξ_ALT · √t
```

где `t` - время в годах с начала термокарстового процесса.

**Применение:**
- Прогноз развития термокарстовых озер и аласов
- Многолетние симуляции (10-100 лет)
- Расчет объема просадки

**Типичные значения:**
- 10 лет: ~3-5 м
- 50 лет: ~7-15 м
- 100 лет: ~10-20 м

### Физическое различие

**ALT (сезонное):**
- Обратимый процесс (оттаивание-замерзание)
- Тепловой баланс замыкается за один год
- Глубина стабильна из года в год (при постоянном климате)

**Термокарст (кумулятивное):**
- Необратимый процесс (таяние подземных льдов)
- Тепло накапливается в грунте
- Глубина растет как √t из-за диффузионного характера теплопереноса

### Базовая формула (для ALT)

```
ξ_ALT = √(2λₜ·DDT_annual / (L·ρw·w^α)) · exp(β·(1-V)) · (1 + γ·ln(ΔT/ΔT₀)) · f_moisture
```

где:
- `λₜ` - теплопроводность талого грунта (Вт/(м·К))
- `DDT_annual` - **годовой** градусо-дни тепла (°C·сек/год)
  - `DDT_annual = T_avg · warm_season_days · 86400` (секунды)
  - Для Центральной Якутии: ~2.5°C × 110 дней × 86400 с ≈ 23.8 млн °C·с
- `L` - скрытая теплота плавления льда (334,000 Дж/кг)
- `ρw` - плотность воды (1000 кг/м³)
- `w` - объемная льдистость (0-1)
- `α` - показатель степени льдистости (0.7 для Якутии, см. ниже)
- `β` - коэффициент покрова (0.30 для Якутии)
- `V` - плотность растительного покрова (0-1)
- `γ` - коэффициент континентальности (0.12 для Якутии)
- `ΔT` - годовая амплитуда температур (°C)
- `ΔT₀` - базовая амплитуда (40°C)
- `f_moisture` - фактор влажности грунта (см. ниже)

## Выбор показателя степени льдистости (α)

### Проблема

Классическая формула Стефана использует `w^1.0` (линейная зависимость от льдистости). Однако эмпирические данные показывают систематическое завышение глубины протаивания на 10-20% для высокольдистых грунтов Якутии.

### Два подхода

#### Подход A: w^1.0 (классический Стефан)

**Физическое обоснование:**
- Прямо следует из уравнения теплового баланса
- Вся скрытая теплота идет на плавление льда пропорционально его содержанию

**Результаты валидации (ALT, t=1 год):**
| Участок | Модель (w^1.0, β=0.45) | Наблюдение | Отклонение |
|---------|------------------------|------------|------------|
| Neleger лес | 1.46 м | 1.0-1.3 м | +12-46% |
| Churapcha лес | 1.56 м | ~1.3 м | +20% |
| Churapcha луг | 2.48 м | ~2.0 м | +24% |

**Вывод:** Систематическое завышение 12-24%

#### Подход B: w^0.7 (эмпирическая поправка)

**Физическое обоснование:**
- При высокой льдистости часть тепла поглощается латентной теплотой раньше, чем вносит вклад в протаивание
- Нелинейный эффект в переходной зоне (Dall'Amico et al., 2011)
- Неоднородное распределение льда в грунте создает эффективную льдистость ниже номинальной

**Результаты валидации (ALT, t=1 год):**
| Участок | Модель (w^0.7, β=0.30) | Наблюдение | Отклонение |
|---------|------------------------|------------|------------|
| Neleger лес | 1.30 м | 1.0-1.3 м | 0-30% ✅ |
| Churapcha лес | 1.40 м | ~1.3 м | +8% ✅ |
| Churapcha луг | 2.01 м | ~2.0 м | +0.5% ✅✅ |

**Вывод:** Отличное совпадение с наблюдениями

### Компенсация параметров

**Важно:** При переходе с `w^1.0` на `w^0.7` необходимо скорректировать `β`:
- `w^1.0` требует `β = 0.45` (сильное влияние растительности)
- `w^0.7` требует `β = 0.30` (ослабленное влияние растительности)

Эти параметры взаимно компенсируют друг друга. Нельзя использовать `w^0.7` с `β = 0.45` или наоборот.

### Текущий выбор: w^0.7

**Версия:** 0.2.0  
**Дата решения:** 2026-04-08

**Обоснование:**
1. Лучшее совпадение с полевыми данными (отклонение <10% vs 12-24%)
2. Физически интерпретируемо через нелинейные эффекты в переходной зоне
3. Согласуется с работами по моделированию мерзлых грунтов (Dall'Amico 2011)
4. Калибровано по данным Churapcha и Neleger

**Ограничения:**
- Это эмпирическая поправка, не выведенная из первых принципов
- Показатель 0.7 получен подгонкой под данные Якутии
- Может требовать корректировки для других регионов
- Требует совместной калибровки с β

## Фактор влажности грунта (f_moisture)

### Текущая реализация (v0.2.0)

```
f_moisture = 1.0 + 0.3 · water_availability
```

где `water_availability` - доступность воды в грунте (0-1).

### Проблема: неверная архитектура

Это не просто необоснованный коэффициент - это **архитектурная ошибка**. Влажность уже имеет правильное место в формуле - внутри `λₜ` (теплопроводности).

**Физика:**
- Влажный грунт имеет большую теплопроводность → `λₜ` больше → `ξ` больше
- Теплопроводность доминирует над теплоёмкостью ([TRB 1948](https://onlinepubs.trb.org/Onlinepubs/hrbproceedings/28/28-035.pdf), [ResearchGate 2023](https://www.researchgate.net/publication/373088373))
- Влажность грунта имеет наибольшее влияние на теплопроводность среди всех параметров

**Правильная модель - Йоханзен (1975):**

```
λₜ(Sr) = λdry + (λsat - λdry) · Sr^0.7
```

где:
- `Sr` - степень насыщения (saturation ratio, 0-1)
- `λdry` - теплопроводность сухого грунта
- `λsat` - теплопроводность насыщенного грунта

**Численный пример (суглинок):**
- Сухой: `λdry ≈ 0.5` Вт/(м·К)
- Насыщенный: `λsat ≈ 1.8` Вт/(м·К)
- Диапазон влияния: 0.62x - 1.18x от базового значения

**Текущая проблема:**
- `f_moisture ∈ [1.0, 1.3]` численно близок к правильному диапазону
- Но он умножает поверх уже посчитанного `λₜ`
- Это **двойной учёт** если `λₜ` уже зависит от влажности

### Рекомендуемые изменения для v0.3.0

1. **Убрать `f_moisture` из формулы Атласова**
   ```
   ξ_ALT = √(2λₜ·DDT_annual / (L·ρw·w^α)) · exp(β·(1-V)) · (1 + γ·ln(ΔT/ΔT₀))
   ```

2. **Сделать `λₜ` функцией влажности по Йоханзену**
   ```rust
   fn thermal_conductivity(&self, saturation_ratio: f64) -> f64 {
       let lambda_dry = self.soil_type.lambda_dry();
       let lambda_sat = self.soil_type.lambda_sat();
       lambda_dry + (lambda_sat - lambda_dry) * saturation_ratio.powf(0.7)
   }
   ```

3. **Переименовать параметр для ясности**
   - Было: `water_availability` (неясный смысл)
   - Стало: `soil_saturation_ratio` или `Sr` (физически очевидно)

4. **Обновить типы грунтов**
   ```rust
   pub enum SoilType {
       Sand { lambda_dry: 0.3, lambda_sat: 2.2 },
       Silt { lambda_dry: 0.5, lambda_sat: 1.8 },
       Clay { lambda_dry: 0.4, lambda_sat: 1.6 },
       Peat { lambda_dry: 0.06, lambda_sat: 0.5 },
   }
   ```

### Временное решение (v0.2.0)

В текущей реализации используется фиксированное значение `water_availability = 0.3` (средняя влажность), что даёт `f_moisture = 1.09`. Это добавляет ~9% к глубине протаивания. Численно близко к правильному эффекту, но архитектурно неверно.

## Обратная формула

При использовании `w^α` в прямой формуле, обратная формула для определения возраста должна использовать тот же показатель.

### Для сезонного ALT (не применимо)

Сезонная глубина не зависит от возраста - она стабильна из года в год.

### Для кумулятивного термокарста

Из формулы `ξ_thermo(t) = ξ_ALT · √t` следует:

```
t = (ξ_thermo / ξ_ALT)²
```

где `ξ_ALT` рассчитывается по базовой формуле для данного участка.

**Практическое применение:**
1. Измерить текущую глубину термокарста `ξ_thermo`
2. Рассчитать `ξ_ALT` для данного участка по климатическим параметрам
3. Определить возраст: `t = (ξ_thermo / ξ_ALT)²`

**Пример:**
- Измеренная глубина термокарстового озера: 10 м
- Расчетный ALT для этого участка: 1.5 м
- Возраст: t = (10 / 1.5)² ≈ 44 года

**Критически важно:** Прямая и обратная формулы должны использовать одинаковый показатель `α`, иначе они не будут взаимообратными.
