### ⚡ КРИТИЧЕСКИЙ ПРИОРИТЕТ (P0)

#### 1. Фазовые переходы лед ↔ вода
**Статус:** Не реализовано  
**Важность:** 🔴 КРИТИЧНО  
**Сложность:** Высокая  
**Время:** 2-3 недели

**Почему это №1:**
- Термокарст = процесс таяния льда
- Без фазовых переходов модель неполная
- Скрытая теплота плавления - ключевой фактор
- Влияет на все остальные процессы

**Что нужно реализовать:**

```rust
// crates/physics/src/phase_change.rs

pub struct PhaseChangeModel {
    /// Температура фазового перехода (°C)
    pub melting_point: f64,
    /// Скрытая теплота плавления (Дж/кг)
    pub latent_heat: f64,
    /// Ширина переходной зоны (°C)
    pub transition_width: f64,
}

impl PhaseChangeModel {
    /// Доля жидкой воды при данной температуре
    /// θ_l(T) = 0.5 * (1 + tanh((T - T_m) / ΔT))
    pub fn liquid_fraction(&self, temperature: f64) -> f64;
    
    /// Эффективная теплоемкость с учетом фазового перехода
    /// C_eff = C + L * dθ_l/dT
    pub fn effective_heat_capacity(&self, temperature: f64) -> f64;
    
    /// Скорость таяния/замерзания (кг/(м³·с))
    pub fn phase_change_rate(&self, temperature: f64, heat_flux: f64) -> f64;
}
```

**Интеграция:**
- Связать с `ThawDepthCalculator`
- Обновить `HeatTransfer2D` для учета фазовых переходов
- Добавить в `SimulationEngine`

**Валидация:**
- Stefan problem (аналитическое решение)
- Эксперименты по таянию льда
- Данные из Якутии (скорость протаивания)

**Метрики успеха:**
- Точность ±5% на Stefan problem
- Реалистичные скорости таяния (1-10 см/день)
- Корректный учет скрытой теплоты

### 🔥 ВЫСОКИЙ ПРИОРИТЕТ (P1)

#### 2. Гидрология (потоки воды)
**Статус:** Частично (water_availability)  
**Важность:** 🟠 ВЫСОКАЯ  
**Сложность:** Очень высокая  
**Время:** 3-4 недели

**Почему это важно:**
- Вода = главный драйвер термокарста
- Определяет скорость расширения
- Влияет на теплоперенос
- Необходимо для реалистичных прогнозов

**Что нужно реализовать:**

```rust
// crates/physics/src/hydrology.rs

pub struct HydrologicalModel {
    /// Гидравлическая проводимость (м/с)
    pub hydraulic_conductivity: f64,
    /// Пористость
    pub porosity: f64,
    /// Насыщенность водой (0-1)
    pub saturation: f64,
}

impl HydrologicalModel {
    /// Закон Дарси: q = -K * ∇h
    pub fn darcy_flux(&self, hydraulic_gradient: f64) -> f64;
    
    /// Уравнение Ричардса (ненасыщенный поток)
    pub fn richards_equation(&self, pressure_head: f64) -> f64;
    
    /// Инфильтрация осадков
    pub fn infiltration_rate(&self, precipitation: f64) -> f64;
    
    /// Испарение с поверхности
    pub fn evaporation_rate(&self, temperature: f64, humidity: f64) -> f64;
    
    /// Аккумуляция воды в термокарсте
    pub fn water_accumulation(&self, inflow: f64, outflow: f64) -> f64;
}
```

**Компоненты:**
1. **Поверхностный сток**: осадки → инфильтрация
2. **Подземный поток**: закон Дарси в мерзлом грунте
3. **Капиллярный подъем**: из талика
4. **Испарение**: связь с SEB (LE)
5. **Аккумуляция**: формирование озер

**Интеграция:**
- Связать с `SurfaceEnergyBalance` (испарение)
- Обновить `LateralExpansionCalculator` (вода ускоряет расширение)
- Добавить в 2D модель (латеральные потоки)

**Валидация:**
- Данные по уровню воды в термокарстовых озерах
- Скорости инфильтрации в мерзлых грунтах
- Баланс воды (осадки - испарение - сток)

**Метрики успеха:**
- Реалистичные скорости потока (10⁻⁶ - 10⁻⁴ м/с)
- Корректная аккумуляция воды
- Баланс воды замыкается (ошибка < 10%)

### 🟡 СРЕДНИЙ ПРИОРИТЕТ (P2)

#### 3. Байесовская калибровка параметров
**Статус:** Есть базовая калибровка  
**Важность:** 🟡 СРЕДНЯЯ  
**Сложность:** Высокая  
**Время:** 2-3 недели

**Почему это нужно:**
- Текущие параметры - экспертные оценки
- Нужна автоматическая подгонка под данные
- Количественная оценка неопределенности параметров
- Улучшит точность прогнозов

**Что нужно реализовать:**

```rust
// crates/simulation/src/bayesian_calibration.rs

pub struct BayesianCalibrator {
    /// Prior распределения параметров
    pub priors: HashMap<String, Distribution>,
    /// Наблюдаемые данные
    pub observations: Vec<Observation>,
    /// MCMC sampler
    pub sampler: MCMCSampler,
}

impl BayesianCalibrator {
    /// Likelihood функция: P(data | params)
    pub fn likelihood(&self, params: &EnvironmentParams) -> f64;
    
    /// Posterior: P(params | data) ∝ P(data | params) * P(params)
    pub fn posterior(&self, params: &EnvironmentParams) -> f64;
    
    /// MCMC sampling (Metropolis-Hastings)
    pub fn mcmc_sample(&mut self, n_iterations: usize) -> Vec<EnvironmentParams>;
    
    /// Оценка параметров (MAP - Maximum A Posteriori)
    pub fn estimate_map(&self) -> EnvironmentParams;
    
    /// Доверительные интервалы параметров
    pub fn parameter_credible_intervals(&self) -> HashMap<String, (f64, f64)>;
}
```

**Методы:**
1. **MCMC (Markov Chain Monte Carlo)**
   - Metropolis-Hastings
   - Adaptive MCMC
   - Parallel tempering

2. **Variational Inference**
   - Быстрее MCMC
   - Приближенный posterior

3. **Particle Filter**
   - Для временных рядов
   - Последовательная ассимиляция данных

**Интеграция:**
- Использовать существующий `UncertaintyAnalyzer`
- Расширить `ModelCalibrator`
- Добавить в CLI: `thermokarst calibrate --method bayesian`

**Валидация:**
- Synthetic data (известные параметры)
- Cross-validation на реальных данных
- Сравнение с экспертными оценками

**Метрики успеха:**
- Posterior сходится (Gelman-Rubin < 1.1)
- Улучшение RMSE на 20-30%
- Реалистичные доверительные интервалы

### 🟢 НИЗКИЙ ПРИОРИТЕТ (P3)

#### 4. Полная связка SEB ↔ грунт
**Статус:** Частично реализовано  
**Важность:** 🟢 ЖЕЛАТЕЛЬНО  
**Сложность:** Средняя  
**Время:** 1-2 недели

**Почему это полезно:**
- Замкнутая система энергии
- Более точные прогнозы температуры
- Учет обратных связей
- Научная полнота модели

**Что нужно реализовать:**

```rust
// crates/physics/src/coupled_model.rs

pub struct CoupledSurfaceGroundModel {
    /// Энергетический баланс поверхности
    pub seb: FullEnergyBalance,
    /// 2D теплоперенос в грунте
    pub ground: HeatTransfer2D,
    /// Гидрология
    pub hydrology: HydrologicalModel,
}

impl CoupledSurfaceGroundModel {
    /// Итерационное решение связанной системы
    pub fn solve_coupled(&mut self, dt: f64) -> Result<()> {
        // 1. SEB → температура поверхности
        let t_surface = self.seb.calculate_surface_temperature(...);
        
        // 2. Граничное условие для грунта
        self.ground.set_surface_temperature(t_surface);
        
        // 3. Теплоперенос в грунте
        self.ground.step()?;
        
        // 4. Поток тепла из грунта → SEB
        let ground_flux = self.ground.surface_heat_flux();
        
        // 5. Обновление SEB с новым потоком
        self.seb.update_ground_flux(ground_flux);
        
        // Итерации до сходимости
        Ok(())
    }
    
    /// Проверка сходимости
    pub fn check_convergence(&self) -> bool;
}
```

**Компоненты:**
1. **Итерационная схема**: SEB ↔ грунт
2. **Граничные условия**: температура/поток на поверхности
3. **Сходимость**: критерий остановки итераций
4. **Временной шаг**: адаптивный dt

**Интеграция:**
- Объединить `FullEnergyBalance` + `HeatTransfer2D`
- Добавить в `SimulationEngine`
- Опциональный режим (для точных расчетов)

**Валидация:**
- Сравнение с раздельными расчетами
- Проверка энергобаланса (замыкание)
- Данные по температурным профилям

**Метрики успеха:**
- Сходимость за 3-5 итераций
- Энергобаланс замыкается (< 1% ошибка)
- Реалистичные температурные профили
